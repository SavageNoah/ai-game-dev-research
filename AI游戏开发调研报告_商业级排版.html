Codex 是当前游戏开发最热门的 AI 编程代理。支持 GPT-5-Codex 模型、沙盒执行、AGENTS.md 自动加载、Skills 系统和 Playwright 浏览器测试。

官方推荐游戏开发流程（7 步）

  1. Plan 模式 — 先让 Codex 写计划，不直接写代码（减少返工 85%）
  2. AGENTS.md — 定义技术栈、编码规范、游戏规则
  3. 可玩循环 — 先做最小可玩单元：移动+跳跃+地面+得分
  4. Playwright 测试 — 在沙盒浏览器中实时验证
  5. UI 调优 — 逐个添加 HUD、菜单、控制
  6. 复杂逻辑 — 自评估循环处理敌人 AI、物理系统

3.2 Game Feel 的 12 个核心要素

关键发现：AI 编程工具默认不会主动实现以下任何一个要素！这些都必须在 AGENTS.md 或 prompt 中明确要求。

移动与跳跃

  • 加速度与摩擦力：MOVE_ACCEL=1400, GROUND_DRAG=1200, AIR_DRAG=200
  • 土狼时间：离开平台后 90ms 内仍可跳跃
  • 跳跃缓冲：落地前 100ms 按跳跃仍有效
  • 可变高度跳跃：按住跳高，轻点跳低（松开速度减半）

视觉与打击感

  • 屏幕震动：踩踏/受击时画面抖动，指数衰减
  • 粒子效果：跳跃扬尘、着陆冲击、收集闪光
  • 挤压拉伸：起跳压扁、空中拉长（迪士尼原则 #1）
  • 命中暂停：攻击命中冻结 3-5 帧增加冲击感

相机与感知

  • 相机延迟：lerp=0.08 平滑跟随
  • 相机死区：60×40px 内不跟随
  • 音效分层：跳跃/着陆/收集各有独立音效
  • 视觉提示：收集品浮动+发光

第六章　教育游戏设计：英语学习 × 游戏融合

6.1 游戏化教育理论基础

ScienceDirect 系统性综述（21 项实证研究）验证了游戏化外语学习的有效性：

  • 即时反馈是最有效的学习驱动因素
  • 游戏元素（XP、排行榜、徽章）显著提升学习动力
  • 间隔重复结合游戏机制可提升记忆保留率 30%+

6.2 Duolingo 八核驱动框架（Octalysis）

Duolingo 拥有 4.77 亿日活用户，其成功归功于 Octalysis 八核驱动：

核心驱动Duolingo 策略可借鉴到英语游戏

6.3 三套融合设计方案

方案 A：单词跳跃（Word Jumper）

玩家在平台上跳跃收集字母，按正确顺序拼出单词才能过关。

  • 每关一组目标单词（如 10 个水果类词汇）
  • 字母随机散布，需按正确顺序收集
  • Boss 战 = 看中文释义，跳到正确英文单词平台
  • 每关结束后显示单词本，可复习

方案 B：语法赛车（Grammar Racer）

赛道分叉为 2-3 条路，每条标注不同语法选项。选对加速，选错减速+弹出解析。

  • 例如：路面"I ___ happy"，三条路 am/is/are
  • 每圈 5-10 个语法挑战
  • 竞速模式：好友 PK

方案 C：单词射击（Word Shooter）

自动横向卷轴，天空掉落英文单词气球，底部显示中文释义，射击正确单词。

  • 同时出现 3-5 个单词气球
  • 难度递进：单词更难、速度更快、干扰更多

第九章　AI 工具高级使用技巧

9.1 AGENTS.md 最佳实践

# AGENTS.md ## 项目：英语学习平台跳跃游戏 ## 引擎：Phaser 3.80 + TypeScript ## 构建：Vite ## 编码规范 - 所有物理常量集中在 src/config/constants.ts - 使用状态机模式管理角色状态 - 每个Entity独立一个文件 - 禁止使用 any 类型 ## 游戏特定规则 - 角色跳跃必须实现土狼时间(90ms)和跳跃缓冲(100ms) - 相机使用 lerp=0.08 的平滑跟随 - 踩踏敌人触发屏幕震动(强度0.01, 持续100ms) - 错误答案不直接结束游戏，弹出教学卡片 - 每关 6-10 个目标单词 ## 测试要求 - pnpm test (Playwright 测试) - 修改物理参数后需手动测试跳跃手感

9.2 核心 Prompt 示例

角色控制器 Prompt

使用 Phaser 3 + TypeScript 创建平台跳跃角色控制器。 要求： 1. 基于加速度的运动模型（不是直接设velocity） 2. MOVE_ACCEL=1400, GROUND_DRAG=1200, AIR_DRAG=200 3. 可变高度跳跃（按住跳得高，轻点跳得低） 4. 土狼时间 90ms 5. 跳跃缓冲 100ms 6. 最大水平速度 220px/s 7. 角色碰撞框 16x26，偏移(8,4) 参考 src/config/constants.ts 中的参数定义。 遵循 AGENTS.md 中的编码规范。

教学卡片系统 Prompt

创建英语教学卡片弹窗系统。 当玩家收集字母顺序错误时： 1. 游戏暂停（不结束） 2. 弹出教学卡片：正确拼写 + 音标 + 中文释义 + 例句 3. 卡片有淡入动画（200ms） 4. 玩家点击"我知道了"后继续游戏 5. 错词记录到复习列表 UI风格：圆角白色卡片，深蓝色标题，灯泡图标

9.3 30 条黄金法则摘要（Indie Hackers）

规划

  • 先用 ChatGPT/Grok 写计划再让 Cursor 执行（减幻觉 85%）
  • 生成多个初始版本，选最好的
  • 设置 Project Rules 统一 AI 行为

开发

  • 每次只做一个小的局部改动
  • 频繁 commit 创建回滚点
  • 图片描述画面比文字有效 10 倍

调试

  • 调试超 2 小时就回退重写 prompt
  • 让 AI 写单元测试和集成测试
  • 保留"接下来 10 个 prompt"列表
